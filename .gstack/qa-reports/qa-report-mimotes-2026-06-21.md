## Issues Found

### ISSUE-001 — Login fails silently with wrong password
- **Severity:** HIGH
- **Category:** Functional
- **Page:** /login
- **Description:** When entering wrong password, the login form does not show any error message. The form just stays on the same page with no feedback to the user.
- **Expected:** Error message like "Email atau password salah" should appear.
- **Actual:** Silent failure — user doesn't know if credentials are wrong or something else broke.
- **Repro steps:**
  1. Go to /login
  2. Enter valid email (admin@mimotes.com) with wrong password
  3. Click "Masuk"
  4. Page stays on /login with no error message
- **Impact:** Users cannot diagnose login failures. High friction for first-time users.
- **Screenshot:** screenshots/login-after-submit.png

### ISSUE-002 — CSS preload warning (unused resource)
- **Severity:** MEDIUM
- **Category:** Performance
- **Page:** /dashboard
- **Description:** Console warning: "The resource ...css was preloaded using link preload but not used within a few seconds from the window's load event."
- **Impact:** Minor performance waste — browser preloads a CSS file that isn't used. Not user-facing but affects Lighthouse score.
- **Fix:** Remove the preload link or ensure the CSS is actually used on the dashboard page.

### ISSUE-003 — Duplicate conversations in sidebar
- **Severity:** LOW
- **Category:** UX
- **Page:** /chat
- **Description:** The conversation sidebar shows many duplicate entries (e.g., "Ada rumah 100 juta?" appears 6 times, "Apa saja dokumen yang tersedia?" appears 4 times). This clutters the sidebar and makes it hard to find specific conversations.
- **Expected:** Each conversation should appear once, or duplicates should be collapsed/merged.
- **Actual:** Every test query creates a new session even if it's the same question.
- **Impact:** Poor UX for users with many conversations. Sidebar becomes unusable after ~20 sessions.
- **Screenshot:** screenshots/chat-page.png
